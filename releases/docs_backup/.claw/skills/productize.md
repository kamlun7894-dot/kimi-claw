# Skill: /productize

## 描述
將學習/實驗結果轉化為可銷售產品（skill packs, templates, blog posts, case studies）。

## 觸發條件
- 「產品化」「打包」「發佈」
- 完成一次有意義的進化後
- 定期整理（每週）

## 執行步驟

### Step 1: 識別產品機會
檢查最近的進化記錄：
- 有咩值得分享的發現？
- 有咩可重用的 pattern？
- 有咩實驗結果有價值？

### Step 2: 選擇產品形式

| 發現類型 | 產品形式 | 平台 |
|---------|---------|------|
| 新 skill | Skill Pack | ClawHub, Gumroad |
| 有效 prompt | Template Library | Gumroad |
| 實驗過程 | Blog Post | 自有網站 |
| 重大突破 | Case Study | Gumroad |
| 定期更新 | Newsletter | Substack |
| 數據洞察 | 進化報告 | Gumroad |

### Step 3: 打包

**Skill Pack：**
- 整理相關 skills
- 建立 README.md（用途、安裝、示例）
- 建立定價（免費 / $5 / $15 / $30）

**Prompt Template：**
- 整理有效的 prompts
- 分類（writing, coding, analysis）
- 加入使用說明

**Blog Post：**
- 改寫實驗報告成繁中
- 加入故事性
- 加入可執行建議

### Step 4: 準備發佈
- 檢查質量
- 準備截圖/示例
- 寫產品描述

### Step 5: 上架
- 上傳到相應平台
- 記錄產品 URL
- 設置價格

### Step 6: 推廣
- 發佈到社交媒體（如果適合）
- 記錄到產品目錄

## 輸出
- 可銷售產品（在 products/）
- 上架記錄
- 推廣素材

## 產品目錄
維護 `products/catalog.md`：
```markdown
| 產品 | 類型 | 價格 | 平台 | 狀態 | 收入 |
|------|------|------|------|------|------|
| ... | ... | ... | ... | ... | ... |
```
