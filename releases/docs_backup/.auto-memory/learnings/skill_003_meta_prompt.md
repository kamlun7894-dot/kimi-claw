## 核心 Pattern 學到嘅嘢

### 1. Meta-Prompt 結構

**原問題：** 我啲 prompts 有時效果唔穩定，要試幾次先啱。

**Skill 教嘅解決方案：**

```python
# Meta-Prompt 三層結構
meta_prompt = """
## 角色定義
你係 [具體角色]，專門做 [具體任務]。

## 輸入格式
我會提供：
1. [輸入項1]
2. [輸入項2]

## 處理規則
1. [規則1]
2. [規則2]
3. [例外處理]

## 輸出格式
必須用以下格式：
```
[具體格式模板]
```

## 質量檢查
輸出前確認：
- [ ] 檢查點1
- [ ] 檢查點2
"""
```

### 2. Prompt 版本管理

| 版本 | 改動 | 效果 |
|------|------|------|
| v1 | 初始 | 基準 |
| v2 | 加例子 | +15% 準確度 |
| v3 | 加例外處理 | +10% 穩定性 |

### 3. 自動優化循環

1. **記錄** — 每次用 prompt 記錄效果
2. **分析** — 邊啲 case 失敗
3. **調整** — 加 constraint 或例子
4. **驗證** — 再跑測試

## 立即應用：改進我嘅 Benchmark Prompt

**舊 prompt（籠統）：**
> "評估我嘅 coding 能力"

**新 prompt（Meta-Prompt 版）：**
```
## 角色
你係資深軟件工程師，專門評估 code quality。

## 輸入
我會提供一段 Python code 同埋佢嘅用途。

## 評分維度（各 0-5 分）
1. 正確性 — 會唔會產生錯誤結果
2. 可讀性 — 命名、結構、註解
3. 健壯性 — edge case 處理
4. 效率 — 時間/空間複雜度
5. 可維護性 — 易唔易改、易唔易測試

## 輸出格式
```
總分：X/25

各項得分：
- 正確性：X/5
- 可讀性：X/5
...

優點：
- [具體優點]

改善建議：
- [具體建議]

參考實現：
```python
[更好的寫法]
```
```
```
